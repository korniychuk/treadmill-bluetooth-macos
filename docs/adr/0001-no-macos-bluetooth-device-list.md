# ADR 0001 — Не показываем дорожку в системном списке Bluetooth macOS

- **Статус:** Принято (2026-07-06)

## Контекст

Дорожка не появляется в системном Bluetooth-меню (System Settings / menu-bar),
где висят AirPods, мышь, наушники. Возник вопрос: сделать так, чтобы появлялась?

Причина отсутствия — by design: тот список показывает устройства, **спаренные
(paired/bonded) на уровне ОС** (Classic HID, аудио). Наш демон открывает дорожку
как **BLE GATT-периферию через CoreBluetooth** на уровне приложения, без OS-level
pairing — FTMS читаем/пишем без шифрования и bonding. Так же ведут себя другие
app-managed BLE-устройства (пульсометры, велодатчики). Затащить в список можно
только форсируя bonding (обращение к зашифрованной характеристике), что хрупко и
не даёт ничего полезного: у macOS нет профиля FTMS, батареи у AC-устройства нет.

## Решение

**Оставляем как есть.** Не инициируем OS-level pairing/bonding; дорожка остаётся
исключительно app-managed периферией под управлением демона.

## Последствия

- **Плюсы:**
  - Нет контеншена за единственный BLE central. Периферия обслуживает одного
    central за раз (уже конфликт с телефонным Yesoul, см. `daemon.rs`); OS-level
    авто-коннект на wake/boot добавил бы ещё одну сторону, рейсящую демон.
  - Демон остаётся единственным владельцем линка (задача 013); нет системного
    тумблера «disconnect», которым можно случайно отстрелить связь.
- **Минусы / чем закрыто:**
  - Нет «глазом из меню-бара видно, что подключена». Закрыто лучше: `tm status`
    (presence, скорость, тренировки) и status-bar виджет (задача 009). Развитие
    визуального индикатора «connected» — путь через виджет, не через системное
    Bluetooth-меню.
