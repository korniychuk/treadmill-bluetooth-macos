# 017 — Горячая перезагрузка goals-конфига без рестарта демона

## Проблема

Оператор правит `~/.config/treadmill-bluetooth-macos/goals.json` и хочет, чтобы
демон подхватил изменения без перезапуска и без паузы в подсчёте статистики.

Текущее состояние:
- `workout_gap_minutes` — **уже динамический**: читается на каждом read-time
  (`stats`/`status`/`widget`, и `try_apply_default_speed` в задаче 016),
  ретроактивно. Рестарт не нужен.
- **`goals` (пороги шагов)** — грузятся **один раз** при старте демона
  (`daemon::run`, `goals::load_goals()`), поэтому меняются только после рестарта.

Рестарт безопасен (SQLite коммитит по операции, JSONL построчно — ничего не
теряется): `launchctl kickstart -k gui/$UID/…daemon`. Но лучше — авто-подхват.

## Решение — watch по mtime

Демон следит за mtime `goals.json` и перечитывает цели **только когда файл
реально изменился** (не перечитывает и не логирует каждый тик — никакого churn).

- `goals::config_mtime() -> Option<SystemTime>` — mtime резолвнутого пути
  (env-override → `$HOME`-дефолт), `None` если файла нет/недоступен.
- В `stream_with_presence` — тик `CONFIG_RELOAD_INTERVAL` (5 с). На тике:
  сравнить mtime с запомненным; при изменении — `load_goals()` и, если набор
  отличается от текущего, применить и залогировать один раз (INFO).
- `step_goals` в цикле сессии становится `&mut Vec<Goal>` (было `&[Goal]`).
  `celebrate_reached_goals` продолжает брать `&[Goal]`.

Скоуп только активной сессии: цели нужны лишь во время ходьбы (празднование
порогов). Пока лента off — демон idle, цели не используются; новая сессия
(коннект) в первом тике перечитывает свежий файл. Начальный `load_goals()` в
`run()` сохраняется (первый лог при старте).

Латентность подхвата ≤ 5 с. `stat` файла раз в 5 с ничтожен. Битый/удалённый
файл → mtime меняется → перечитывание с фолбэком на дефолты (WARN/INFO из
`load_goals`, ровно раз на изменение).

Отдельная команда `refresh` не нужна: авто-подхват покрывает идеальный сценарий.
Мгновенное применение по-прежнему доступно рестартом (`launchctl kickstart -k`).

## Затронутые файлы

- `src/goals.rs` — `config_mtime()`.
- `src/daemon.rs` — `CONFIG_RELOAD_INTERVAL`, `config_tick` + mtime-watch в
  `stream_with_presence`; `step_goals: &mut Vec<Goal>`; проводка в `run()`.
- `CLAUDE.md` — заметка, что goals-конфиг подхватывается без рестарта.
