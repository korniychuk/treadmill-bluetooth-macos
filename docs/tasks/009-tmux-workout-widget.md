# 009 — tmux workout widget

## Цель

Показать статистику **текущей** тренировки прямо в tmux-статусбаре (Dracula):
время тренировки, шаги, дистанцию. Виджет **появляется/исчезает** сам:

- дорожка **отключена** (нет BLE-связи или демон мёртв/завис) → виджета нет;
- дорожка **на связи** → виджет виден, а его иконка отражает состояние:
  - `walking` — иду по ленте (базовый вид);
  - `paused` — лента остановлена, связь есть;
  - `away` — сошёл с ленты, лента крутится (`AwayWhileRunning`);
  - `unknown` — связь есть, данных ещё нет.

Живёт в двух репозиториях: данные — здесь (`treadmill-bluetooth-macos`),
презентация (иконки/цвета/формат) — в `ankor-dotfiles`.

## Контракт (важно)

Плагин читает статистику **через CLI-команду**, а не из файла — чтобы смена
внутреннего хранилища (схема SQLite/JSON) не ломала tmux-плагин. Стабильный
контракт — это **вывод команды**, а не структура данных на диске.

### `tm widget`

Новая read-only команда (как `status`: **не открывает BLE-адаптер**).

- Печатает **одну TSV-строку** `STATE\tWALKING_S\tSTEPS\tDISTANCE_M`, пока
  дорожка на связи и heartbeat демона свежий;
- печатает **пусто** (exit 0) во всех остальных случаях → Dracula скрывает
  сегмент (механизм `#{?#{==:$script,},, …}`).

Поля:

- `STATE` ∈ `walking | away | paused | unknown` (маппинг из
  `daemon_status.presence_state`);
- `WALKING_S` = `workouts.walking_time_s` последней тренировки —
  **отфильтрованное** время ходьбы (presence-фильтр уже выкинул step-away и
  паузы: это `36m27s`, а не `raw 41m42s`, что печатает `stats`). Само-
  замораживается при не-ходьбе — тогда ничего не кредитуется;
- `STEPS`, `DISTANCE_M` — из последней тренировки (`latest_workout()`).

### Условие видимости

`daemon_status` существует **и** `connected == true` **и** `updated_at` не
старше `WATCHDOG_STALE_THRESHOLD_S` (95 с). Heartbeat демона обновляет
`updated_at` на каждый idle-tick (≤30 с) и каждый telemetry-sample (~1 с), так
что staleness — надёжный сигнал жизни: `daemon_process_alive()`/`launchctl`
**не нужен** (иначе — два спавна процессов на каждый 2-секундный poll).

## Изменения

### `treadmill-bluetooth-macos`

- `store.rs`: `latest_workout()` (`ORDER BY id DESC LIMIT 1`); `busy_timeout(3s)`
  на открытии соединения — 2-секундный poll читает БД, пока демон пишет, и не
  должен ловить `SQLITE_BUSY`.
- `main.rs`: команда `Widget`, `run_widget()`, чистый `widget_state()` под тест.

### `ankor-dotfiles`

- `tmux/treadmill-widget.sh` — презентация: зовёт `tm widget`, парсит TSV,
  рисует **цветной по состоянию** pill (`#[bg=…]` в выводе). Глифы и цвета —
  переменными сверху. Состояние двойное-кодировано: цвет фона + форма иконки
  (walking=зелёный/`run`, paused=жёлтый/`pause`, away=оранжевый/`alert`,
  unknown=серый/`help`).
- `.tmux.conf`: `custom:treadmill.sh` первым в `@dracula-plugins`,
  `@dracula-custom-plugin-colors "gray white"`, `@dracula-refresh-rate 2`.
- `install.sh`: симлинк `tmux/treadmill-widget.sh` →
  `~/.tmux/plugins/tmux/scripts/treadmill.sh` (Dracula ищет custom-скрипты
  только в своём `scripts/`; `git pull` TPM untracked-симлинк не трогает).

## Ключевые решения / ловушки (дорого дались)

- **Hide-when-off через frame-цвет, НЕ через `@dracula-show-empty-plugins false`.**
  Опция `false` оборачивает *каждый* сегмент в `#{?#{==:…},, …#[fg=#hex]…}`, а `#`
  из hex-цветов внутри `#{?…}` рушит парсер формата tmux (эта версия Dracula,
  ce10069 / 2023) — ломаются цвета **всего** бара. Вместо этого frame-цвет
  плагина ставим `gray` (= фон бара `#44475a`): пустой вывод скрипта → сегмент
  сливается с баром и исчезает; при работе скрипт сам красит pill поверх.
- **Стили `#[…]` в выводе скрипта работают** только потому, что при
  `show-empty-plugins=true` Dracula НЕ оборачивает вывод в `#{==}` (стили на
  верхнем уровне формата). При `false` — см. выше.
- **Опции tmux залипают в живом сервере.** `source-file` не снимает опции,
  которых больше нет в файле; после экспериментов пришлось явно вернуть
  `@dracula-show-empty-plugins true`. На свежем старте .tmux.conf корректен.
- **Иконка = MD-глиф, не FA6.** JetBrainsMono Nerd Font не содержит
  `person-running` (U+F70C, FA6) → тофу. Взяли Material Design `run` (U+F070E)
  и др.; заданы UTF-8 байтами через `printf '\xHH'` (портируемо, ASCII на диске).
- **Время = отфильтрованное `walking_time_s`**, не raw wall-clock (см. выше).

## Открытые вопросы / заметки

- «Тройную дистанцию» из ТЗ трактуем как **три метрики** (время/шаги/дистанция),
  подтверждено оператором.
- Idle-connected после тренировки показывает последнюю тренировку (её totals
  заморожены). Приемлемо для v1.
- Powerline-стрелка перед pill'ом статична (цвет frame = фон бара, ~невидима),
  поэтому у pill'а плоский левый край, а не цветная стрелка. Приемлемо.
