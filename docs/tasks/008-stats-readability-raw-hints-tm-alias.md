# 008 — Читаемость `stats`, raw-подсказки, алиас `tm`

## Контекст

Три мелких, но связанных запроса оператора по UX CLI:

1. **Слитный диапазон времени тренировки.** Строка `#1 15:21–15:25: …` читалась
   с трудом — непонятно, что это начало и конец тренировки.
2. **«Сырое» (raw) время и дистанция.** Presence-фильтр вырезает моменты, когда
   лента крутится, но оператор на ней не шагает (см. `daemon::credit_or_hold`):
   `pending` копится только в `Walking` и **отбрасывается** при
   `AwayWhileRunning`/`Paused`. Значит и `walking_time_s`, и `distance_m` в
   `daily_stats`/`workouts` — уже отфильтрованные (только ходьба). Оператор хотел
   видеть рядом «сырую» цифру (до вычета) серым, в скобках. **Да — дистанция
   фильтруется идентично времени** (оба через тот же `pending`).
3. **Алиас `tm`.** Оператор вручную сделал
   `ln -s target/release/treadmill-bluetooth-macos ~/.bin/tm`; попросил, чтобы
   симлинк ставился/снимался install/uninstall-скриптами.

## Решение

### Raw через реконструкцию из `raw_samples` (без новых колонок)

Разделили задачу на два независимых вопроса:

- **Raw-время — арифметика по уже имеющимся полям.** `ended_at − started_at`
  тренировки **и есть** её сырой span: away-разрывы >15 мин разбивают тренировку
  на разные строки (`WORKOUT_GAP_THRESHOLD_S`), поэтому внутри одной строки span
  = ходьба + короткие вырезанные away/pause внутри неё. Хранить нечего.
- **Raw-дистанция — единственное, чего нет в агрегатах**, но она уже пишется
  по-кадрово в `raw_samples.distance_m` (кумулятивный счётчик устройства,
  тег `ts_ms`). `Store::raw_distance_m(started_at, ended_at)` берёт **сумму
  положительных покадровых дельт** в окне тренировки (а не `MAX − MIN`) — так
  цифра остаётся корректной, если дорожку выключили/включили посреди сессии и её
  счётчик сбросился в ноль (наблюдалось: одно окно тренировки покрывает 2 сессии
  с ресетом; live-проверка: сумма-дельт 448 м против MAX−MIN 444 м).

Почему read-path реконструкция, а не forward-only колонки: решающий довод —
**ретроактивность**. Оператор смотрит на *уже записанную сегодняшнюю* тренировку;
forward-only колонки показали бы по ней пусто. Плюс хранение вперёд втащило бы
второй аккумулятор в горячий `credit_or_hold`, который должен **удерживать** raw
на away (обратно фильтрованному `pending`) и попадать в правильную разбитую
тренировку — лишняя поверхность на hot-path. Реконструкция держит всё в
read-path и работает на существующих данных.

### Guard'ы (чтобы не показать мусор)

- `workout_raw()` отдаёт `None` по дистанции/времени, если реконструкция
  «ниже» отфильтрованного значения (raw обязан быть надмножеством ходьбы —
  меньше значит потерянные сэмплы) → подсказка просто опускается.
- **Day-строка остаётся на отфильтрованных цифрах** (raw — только по
  тренировкам). Day-level raw пришлось бы суммировать span'ы тренировок, но
  `daily_stats` может кредитовать активность, не ставшую строкой `workouts`
  (midnight/gap edge-cases из `store`), → сумма молча занижала бы. Точное окно
  `[started_at, ended_at]` есть только у тренировки.

### Формат

- Диапазон: `15:21 → 15:38` (стрелка + разрядка вместо слитного `15:21–15:25:`).
- Raw-подсказка: `0.66 km (raw 0.67)` и `16m18s (raw 17m18s)`.
- **Серый = ANSI dim (`\x1b[2m…\x1b[0m`), только на TTY** (`std::io::IsTerminal`),
  чтобы `tm stats | grep` не ловил ESC-последовательности. Прежний код был
  голым `println!` — цветового пути не было, завели с нуля.

Пример:

```
2026-07-05: 1653 steps, 0.67 km, 17m51s walking
  #1  15:21 → 15:38   1605 steps, 0.66 km (raw 0.67), 16m18s (raw 17m18s)
```

### Алиас `tm` в скриптах

- `install-daemon.sh`: `ln -sfn` на release-бинарь в `$LINK_DIR` (default
  `~/.bin`), имя `$LINK_NAME` (default `tm`). `ln -sfn` заменяет **наш** stale
  симлинк на месте, но обычный файл с тем же именем не трогается
  (`[[ -e && ! -L ]]` → warn + skip). Если `LINK_DIR` не в `PATH` — подсказка.
  `LINK_NAME=""` отключает.
- `uninstall-daemon.sh`: снимает симлинк, **только если** это симлинк на наш
  `BIN_NAME` (`readlink` заканчивается на `/treadmill-bluetooth-macos`) — чужой
  `tm` не трогаем.
- Симлинк указывает на артефакт сборки → подхватывает свежий бинарь после
  каждого rebuild, отдельного шага обновления не нужно.

## Верификация

- `cargo build --release`, `cargo test` (23/23), clippy — только 3 известных
  dead-code из 006.
- `tm stats` живьём: диапазон со стрелкой, raw-подсказки; в пайпе ESC-кодов нет
  (TTY-guard).
- Логика симлинка прогнана изолированно (5 кейсов: fresh create, идемпотентная
  замена своего симлинка, отказ затирать обычный файл, uninstall снимает свой
  симлинк, uninstall не трогает чужой).
- **Install-скрипт целиком намеренно не запускался**: оператор в этот момент шёл
  (данные обновлялись), а reload LaunchAgent прервал бы трекинг посреди
  тренировки. Симлинк подхватится при следующем штатном `install-daemon.sh`;
  ручной симлинк оператора уже указывает на тот же бинарь.
