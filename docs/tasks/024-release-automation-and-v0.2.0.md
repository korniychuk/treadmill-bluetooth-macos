# 024 — Скрипт релиза одной командой + выпуск v0.2.0

## Проблема

Релиз тег-триггерный (`release.yml`: `on: push: tags: v*`). После v0.1.0 в `main`
ушло 9 коммитов (задачи 020–023), но тег не ставили → GitHub-релиза нет. Шаги
релиза (бамп версии, дата в CHANGELOG, коммит, тег, пуш, следить за workflow)
делаются руками и **забываются**. Нужна одна команда.

## Решение — `scripts/release.sh <version>`

Догфудим на выпуске **v0.2.0**.

### Что делает скрипт

`bash scripts/release.sh 0.2.0` (Bash 5, `set -euo pipefail`):

1. **Валидация** (fail-fast, чтобы не выпустить кривой релиз):
   - версия формата `X.Y.Z` (ведущий `v` срезается, если передан);
   - на ветке `main`;
   - рабочее дерево чистое (иначе — «закоммить/стешни»);
   - тег `vX.Y.Z` ещё не существует;
   - секция `## [Unreleased]` в CHANGELOG **не пустая** (иначе «нечего релизить —
     сперва заполни Unreleased»).
2. **Бамп версии** в `Cargo.toml` (первый `version = "…"` = пакетный) + `cargo
   build` для синка `Cargo.lock`.
3. **CHANGELOG**: вставить `## [X.Y.Z] — YYYY-MM-DD` сразу под `## [Unreleased]` —
   контент из Unreleased уезжает под датированный заголовок, Unreleased снова
   пустой на будущее (Keep a Changelog).
4. **Коммит** `chore(release): vX.Y.Z` (явный pathspec: Cargo.toml/Cargo.lock/
   CHANGELOG.md), **аннотированный тег** `vX.Y.Z`, **пуш** ветки + тега.
5. **Проследить**: если есть `gh` — дождаться прогона `release.yml` и показать URL
   релиза; иначе напечатать команду.

Ручной остаётся только **проза CHANGELOG**: `[Unreleased]` заполняется по ходу
работы (или перед релизом) — это человеческое суждение, скрипт лишь датирует.

### CHANGELOG под v0.2.0 (заполняется перед прогоном)

- **Added**: авто-пауза простаивающей ленты (020); показ загруженного конфига в
  `tm status` (022).
- **Changed**: конфиг теперь TOML `config.toml` (021 rename + 023 TOML) — ради
  комментируемых дефолтов; hot-reload/резолвинг переописаны.
- **Removed**: JSON-конфиг и фолбэки `goals.json`/`TREADMILL_GOALS_CONFIG`.
- **Migration** (breaking для пользователя): `goals.json` → `config.toml`,
  JSON → TOML. По SemVer 0.x breaking = **minor** → `v0.2.0`.

## Затронутые файлы

- `scripts/release.sh` — новый.
- `CHANGELOG.md` — заполнить `[Unreleased]` (перед прогоном; скрипт датирует).
- `CLAUDE.md` — строка про `scripts/release.sh` в командах.
- (скриптом) `Cargo.toml`/`Cargo.lock` — версия; `CHANGELOG.md` — дата.

## Проверка

- Прогнать `bash scripts/release.sh 0.2.0` → тег запушен, `release.yml` зелёный,
  на GitHub появился релиз v0.2.0 с `.tar.gz`-артефактом.
- Идемпотентность валидации: повторный запуск с тем же тегом падает на «tag exists».
