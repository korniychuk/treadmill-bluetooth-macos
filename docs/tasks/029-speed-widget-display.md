# 029 — Виджет: текущая скорость ленты (км/ч)

**Статус:** implemented и проверено на живом железе

## Найденный по ходу баг (не мой первый заход, но моя вина — не проверил вживую)

Первая версия ломала виджет целиком (не только скорость): `IFS=$'\t' read -r
... <<<"$line"` — таб входит в тройку POSIX "IFS whitespace" (space/tab/
newline), и когда `IFS` состоит **только** из таких символов, `read`
схлопывает **серии** разделителей и обрезает их по краям — как дефолтный
`IFS` с пробелами. Любое пустое поле в середине (например пустой `hr_zone`
перед непустым `speed_kmh`) склеивалось с соседним, и все поля после
сдвигались на одну позицию влево. Это скрытая проблема ещё с задач 026/027 —
раньше маскировалась тем, что хвост после `hr_zone` был либо весь
числовым, либо весь пустым; задача 029 добавила непустое поле сразу после
частенько-пустого `hr_zone`, и баг стал видимым (скорость молча пропадала
из виджета, хотя `tm widget` в консоли честно отдавал `3.1kmh`).

Фикс — заменить табы на ASCII Unit Separator (`0x1F`, не входит в класс
"blank") перед `read`, чтобы каждый разделитель трактовался буквально:
`IFS=$'\x1f' read -r ... <<<"${line//$'\t'/$'\x1f'}"`.
**Дата:** 2026-07-09
**Зависит от:** 025/026 (тот же снапшот-паттерн `daemon_status`, что HR/battery)

## Цель

Показывать текущую скорость ленты в tmux-виджете, в единицах км/ч, как
текстовую нотацию (не иконка — набор нейтральных Nerd Font иконок скорости
не читался однозначно, решение оператора): `3.1kmh`, число — обычным цветом
(текущий `fg`/`STEPS_FG`), суффикс `kmh` — приглушённым (`dim`). Отображение
включается/выключается через config **и** отдельную CLI-команду.

## Формат числа

- Округление до 1 знака после запятой, обычное half-up.
- Если дробная часть после округления — `0`, выводим без точки и без цифры
  после неё (целое число): `3kmh`, а не `3.0kmh`.
- Иначе — с одной цифрой: `3.1kmh`.

## Источник данных

Живая скорость приходит в `data.speed_kmh: Option<f32>` в `src/daemon.rs`
(телеметрия FTMS, `~1с`) — там же, где уже обновляется `speed_history`/
`last_walking_speed` для restore-after-pause (задача 012). Мимо
`zone_hold_last_speed` (тот привязан к Zone Hold и `None`, когда контроллер
не активен) — заводим отдельную пару полей, зеркалируя `last_bpm`/
`last_bpm_ts` (задача 025):

- `DaemonState.last_speed_kmh: Option<f64>` + `last_speed_ts: Option<i64>`
  (unix millis) — обновляются на каждом сэмпле с `Some(speed)`.
- `store::DaemonStatus` — те же два поля, ALTER-колонки в `daemon_status`
  (`REAL`/`INTEGER`), upsert/read как остальной снапшот.
- `tm widget` — 12-е TSV-поле `speed_kmh`: пусто, если конфиг выключен, нет
  свежего сэмпла (freshness-порог как у HR — `WATCHDOG_STALE_THRESHOLD_S`),
  или сама скорость `0` (лента стоит — не показывать `0kmh`, это шум).
  Форматирование (округление/схлопывание `.0`) делается на стороне Rust
  (`widget_speed_field`), tmux-скрипт только красит готовую строку.

## Конфиг

Top-level булев ключ `show_speed` в `config.toml` (рядом с
`workout_gap_minutes`/`auto_pause_minutes`, не отдельная секция — как и они,
простой флаг без вложенности). Дефолт — `false` (opt-in, не захламляем
виджет по умолчанию). Паттерн загрузки — 1:1 с `load_workout_gap_minutes`/
`load_auto_pause` (`goals.rs`): 3-вариантный enum `Configured/Invalid/Unset`,
тихий `Unset`/отсутствующий файл, `WARN` на `Invalid`.

## CLI

`tm speed-widget on/off` (без аргумента — статус: включено/выключено).
**Не** `tm speed` — та команда уже занята (`Speed { kmh: f32 }`, задание
целевой скорости ленты через FTMS Control Point) — коллизия имён отловлена
при планировании, отдельное имя во избежание путаницы. Пишет/читает тот же
top-level `show_speed` — новый маленький хелпер `upsert_top_level_key` в
`goals.rs` (line-based upsert как `zone_hold::upsert_zone_hold_keys`, но без
`[section]`-контекста: ищет/вставляет ключ до первой `[section]`-строки).

## Виджет: как выглядит

Обычно (выключено в конфиге, нет данных, или скорость `0`) — без изменений.
Включено и лента движется:

```
  3.1kmh
```

(число — текущий `fg`/`STEPS_FG`, `kmh` — `dim`/`DIM_UNKNOWN` по состоянию,
как день-тотал у остальных метрик).

## Тесты

- `goals.rs`: unit-тесты на `read_show_speed`/`load_show_speed` (три ветки:
  absent/invalid/configured), по образцу существующих для gap/auto-pause.
- `main.rs`: unit-тест на форматтер `format_speed_kmh` (округление,
  схлопывание `.0`, `0` → пусто).
- `store.rs`: roundtrip `last_speed_kmh`/`last_speed_ts` в `daemon_status`.
